# Governance

This document describes how the project is maintained and how decisions are made.

## Maintainers

Maintainers are responsible for:

- triaging issues and pull requests
- reviewing and merging changes
- maintaining release quality and security posture
- evolving roadmap and architecture

## Decision making

- Minor changes are decided in pull request review.
- Significant changes should start as an Issue or Discussion and seek maintainer alignment before implementation.
- In case of disagreement, maintainers make the final decision to keep project direction coherent.

## Contribution model

The project accepts community pull requests.

Contributors should follow:

- [CONTRIBUTING.md](./CONTRIBUTING.md)
- [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)
- [SECURITY.md](./SECURITY.md)

## Release and roadmap

- Planned direction is tracked in [ROADMAP.md](./ROADMAP.md).
- Notable shipped changes are tracked in [release-notes.md](./release-notes.md).

## Maintainer updates

Maintainer membership may evolve as project needs change.
